推送通知处理

1 我们如何收集和使用您的个人信息：
推送通知接收（消息提醒），当您的 Home Assistant 服务器向您发送推送通知时，为了在设备上向您展示通知内容，通知通过 Apple Push Notification service（APNs）和 Firebase Cloud Messaging 投递至您的设备。通知**标题、消息内容和附加数据**由您的 Home Assistant 服务器生成并推送。这些数据属于您自行托管服务器的内容数据。采集频率：由您的 Home Assistant 服务器触发。处理方式说明：仅本地展示——通知内容展示后不持久保留。

通知附件下载，当推送通知包含媒体附件（如摄像头快照）时，为在通知中展示该附件，通知服务扩展会从您的 Home Assistant 服务器下载图片或视频。这些媒体内容为**敏感个人信息**（可能包含个人或家庭图像）。下载完成后仅临时展示于通知中，自动清理。采集频率：推送通知包含附件时触发。

摄像头推送命令，当推送通知包含摄像头实体 ID 时，为在应用内悬浮层展示摄像头直播画面，应用会解析通知负载中的摄像头标识，并从您的 Home Assistant 服务器获取实时视频流。

信息亭远程管理命令，当推送通知包含信息亭命令时，为执行远程管理操作，应用会解析命令参数并设置**屏幕亮度、系统音量**等设备设置。采集频率：收到对应推送命令时触发。

Live Activity 更新，当推送通知类型为 live_activity 时，为更新灵动岛或锁屏状态，应用会解析 Live Activity 更新数据并在系统 UI 上展示。

2 设备权限调用：
通知权限（推送通知）：当您首次启动应用时，本应用将询问通知权限，以便接收 Home Assistant 服务器向您发送的推送通知。如您拒绝授权，将无法收到推送通知提醒。
