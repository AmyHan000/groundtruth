# OpenHarmony DLP Manager 应用隐私数据流分析

## 项目信息

- **应用名称**: DLP Manager (数据泄露防护管理器)
- **源码仓库**: https://gitcode.com/openharmony/applications_dlp_manager
- **Commit Hash**: fd70a47

## 应用概述

DLP Manager 是 OpenHarmony 系统中的数据泄露防护管理器应用，主要功能包括：

1. **文件保护**: 为文件添加 DLP 保护策略，控制文件的访问权限
2. **文件加密分享**: 将文件加密后分享给指定用户
3. **打开 DLP 文件**: 解密并打开受 DLP 保护的文件
4. **账号管理**: 管理域账号和云账号信息
5. **数据上报**: 上报用户行为和故障事件

## 隐私数据流分析

### 1. 文件保护页/保护文件

**功能描述**: 为文件添加 DLP 保护策略，设置文件所有者和访问权限

**收集的数据**:
- 应用信息: 获取 DLP 管理器自身包信息
- 文件信息: 获取文件大小和类型
- 账号信息: 获取当前登录的域账号信息
- 用户ID: 获取用户 ID 用于操作日志

**数据流**:
- 系统API → 应用信息 → 显示
- 文件系统 → 文件信息 → 加密处理
- 系统API → 账号信息 → 显示
- 系统API → 用户ID → 上报

### 2. 文件加密分享页/加密分享

**功能描述**: 将文件加密后分享给指定用户，支持域账号和云账号

**收集的数据**:
- 联系人信息: 获取联系人手机号
- 设备信息: 获取设备类型
- 云账号信息: 获取当前云账号
- 网络信息: 检查网络连接状态
- 文件信息: 获取文件信息用于加密

**数据流**:
- 联系人应用 → 联系人信息 → 显示
- 系统API → 设备信息 → 显示
- 系统API → 云账号信息 → 显示
- 系统API → 网络信息 → 显示
- 文件系统 → 文件信息 → 加密处理

### 3. 打开DLP文件页/打开文件

**功能描述**: 解密并打开受 DLP 保护的文件

**收集的数据**:
- 文件元数据: 解析 DLP 文件元数据
- 应用信息: 验证调用方应用信息
- 解密状态: 管理文件解密状态
- 安全事件: 上报文件打开安全事件

**数据流**:
- 文件系统 → 文件元数据 → 解析
- 系统API → 应用信息 → 验证
- 本地内存 → 解密状态 → 管理
- 系统API → 安全事件 → 上报

### 4. 应用管理页/账号管理

**功能描述**: 管理域账号和授权账号信息

**收集的数据**:
- 域账号信息: 获取当前登录的域账号信息
- 授权账号列表: 获取已授权账号列表
- 用户ID: 获取用户 ID 用于操作日志
- IPC通信: 与其他应用进行跨进程通信
- 本地存储: 本地存储配置和状态信息

**数据流**:
- 系统API → 域账号信息 → 显示
- 本地数据库 → 授权账号列表 → 显示
- 系统API → 用户ID → 上报
- 系统IPC → IPC通信 → 跨进程调用
- 本地存储 → 配置和状态信息 → 读写

### 5. 数据上报页/事件上报

**功能描述**: 上报用户行为和故障事件到系统事件服务

**收集的数据**:
- 行为事件: 上报用户行为事件
- 故障事件: 上报故障事件
- 文件操作事件: 上报文件操作事件

**数据流**:
- 系统API → 行为事件 → 上报
- 系统API → 故障事件 → 上报
- 系统API → 文件操作事件 → 上报

## 权限使用

| 权限名称 | 使用场景 | 授权方式 | 用途 |
|---------|---------|---------|------|
| ohos.permission.GET_BUNDLE_INFO | 获取应用信息 | 预授权 | 获取应用版本信息 |
| ohos.permission.GET_ACCOUNT_INFO | 获取账号信息 | 预授权 | 获取当前登录账号 |
| ohos.permission.READ_CONTACTS | 读取联系人 | 运行时授权 | 获取联系人手机号 |
| ohos.permission.GET_NETWORK_INFO | 获取网络信息 | 预授权 | 检查网络连接 |
| ohos.permission.DLP_FILE_OPERATE | 操作DLP文件 | 预授权 | 打开和解密DLP文件 |
| ohos.permission.HIVIEW_FAULT | 上报故障事件 | 预授权 | 上报故障信息 |
| ohos.permission.HIVIEW_BEHAVIOR | 上报行为事件 | 预授权 | 上报行为数据 |

## 数据处理方式

1. **本地处理**: 大部分数据在本地处理后显示给用户，不上传到服务器
2. **系统事件上报**: 行为和故障事件通过 hiSysEvent 上报到系统事件服务
3. **安全事件上报**: 安全相关事件通过 securityGuard 上报到安全服务
4. **临时存储**: 部分数据（如文件信息、解密状态）临时存储在内存中
5. **持久化存储**: 授权账号列表存储在本地数据库中
6. **IPC通信**: 与其他应用进行跨进程通信
7. **本地存储**: 配置和状态信息存储在本地preferences中

## 隐私保护措施

1. **数据最小化**: 仅收集必要数据，不收集多余信息
2. **本地优先**: 大部分数据在本地处理，不上传到服务器
3. **事件脱敏**: 日志中对敏感信息进行脱敏处理
4. **权限控制**: 通过系统权限机制控制数据访问
5. **加密保护**: 文件内容通过 DLP 加密保护
